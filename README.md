# Airbnb "Anywhere Door" 🚪✨
> **An AI-first reimagining of Airbnb for group travel and zero-friction trip planning.**
> Built as a case study prototype for the **OP'26 Product Competition**.

Traditional travel booking suffers from "comparison paralysis." The average traveler spends hours toggling between dozens of filters, reading fragmented reviews, coordinating budgets in group chats, and manually compiling day-by-day itineraries. 

Inspired by the classic anime *Doraemon*, we built the **Anywhere Door** (*Dokodemo Door*). The concept is simple: *remove all friction between user intent and the travel outcome.* Instead of filling out forms, you state your dream trip in natural language, and the AI handles the rest.

---

## 📖 The Vision: The Doraemon Principle
In the anime, the Anywhere Door transports you to your destination instantly. Our vision is to bring that instant transit to travel planning. 

With a single prompt—like *"A cozy cabin in Manali for 4 friends under ₹15,000/night, with a fireplace and fast Wi-Fi for work"*—the Gemini-powered engine:
1. **Finds perfect property matches** and auto-summarizes guest feedback into clean pros/cons.
2. **Generates a custom day-by-day local itinerary** directly inside your **Trips** tab.
3. **Calculates a detailed per-person split billing sheet** so group booking is stress-free.

## 🚀 Key Engineering & UI Accomplishments

During the development of this prototype, we focused on premium aesthetics and robust engineering:

### 1. Adaptive Dual-Mode Layout
* **Mobile View:** On mobile, the app is wrapped in a native status overlay and bottom navigation bar, simulating a native iOS/Android shell. The Explore subtabs (`Stays`, `Experiences`, `Services`) are sticky, remaining fixed at the top as you scroll.
* **Desktop View:** On screens wider than 768px, the layout expands to a full viewport grid. The top navigation bar is fixed, tab titles are hidden to prevent redundancy (since the active indicators are highlighted on the header), and the Everywhere Door and filters slide out as elegant side drawers.

### 2. Smart Caching & API Cost Optimization
Because third-party APIs can be costly, we built layers of performance optimization to keep operation 100% under the free tiers:
* **Client-Side Cache:** Listings are cached locally via `localStorage` for 24 hours (partitioned by city and country).
* **Image Caching & Type Partitioning:** Image searches from Pexels are cached client-side (`sessionStorage`) and server-side (Node process-level map) to avoid redundant requests. Caches are partitioned by listing type (stays vs. services) to resolve stale photo hits.
* **Server-Side SerpApi Cache:** Hotel API responses are cached locally in `.serpapi-cache.json` for 24 hours.

### 3. Integrated Wishlists & AI Trips
* Wishlisting now works seamlessly on planned trips. Stays, experiences, and services generated by the AI receive stable, unique identifiers (like `stay-ai-...`).
* The root wishlist resolver searches inside AI-generated trips, meaning any planned card you heart in the **Trips** tab displays beautifully in your main **Wishlists** tab.
* Planned price strings (e.g. `"₹300/person"`) are parsed dynamically so they convert correctly across currencies and display `"Free"` for zero-cost activities.

### 4. High-Fidelity PDF Export (`/deck`)
We built an interactive 7-slide strategy presentation directly under `/deck` explaining the business case, success metrics, and mitigations. We optimized the `@media print` rules to disable heavy CSS blur filters and complex radial gradients, ensuring the deck exports instantly as a clean, landscape PDF.

## 🛠️ Technology Stack
* **Framework:** Next.js 15 (App Router), React 19, Lucide Icons
* **Styling:** Vanilla CSS variables for a fluid, premium design system (including glassmorphic drawers, micro-animations, and concentric border-radii).
* **AI & Search:** Google Gemini SDK (`gemini-2.5-flash` / `gemini-3.5-flash`), SerpApi (Google Hotels), Tavily Search.
* **Media Assets:** Pexels API.

## ⚡ Getting Started (Local Run)

1. **Clone the repository:**
   ```bash
   git clone https://github.com/iaryan08/airbnb-anywhere-door.git
   cd airbnb-anywhere-door/airbnb-anywhere-door
   ```

2. **Install dependencies:**
   ```bash
   pnpm install
   ```

3. **Configure Environment Variables:**
   Create a `.env.local` file inside the `airbnb-anywhere-door/` directory:
   ```env
   GEMINI_API_KEY=your_api_key_here
   PEXELS_API_KEY=your_api_key_here
   ```
   *(Note: You can get free keys from Google AI Studio and the Pexels developer page.)*

4. **Launch the app:**
   ```bash
   pnpm run dev
   ```
   Open **[http://localhost:3000](http://localhost:3000)** for the mockup app, or navigate to **`/deck`** to view the strategy presentation slides.

5. **Build and Run Production Server:**
   ```bash
   pnpm run build
   ```
   ```bash
   pnpm run start
   ```
