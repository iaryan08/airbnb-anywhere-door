# Airbnb Anywhere Door — Next.js Application Module

This directory contains the Next.js 15 App Router web application that acts as the interactive prototype for the Airbnb "Anywhere Door" GenAI case study.

---

## 📂 Directory Structure

```
src/
├── app/
│   ├── api/
│   │   ├── geolocation/    # Geolocation lookup proxy (CORS bypass)
│   │   ├── images/         # Pexels photo search fallback API
│   │   └── plan/           # GenAI Travel Planner endpoint (Gemini Flash)
│   ├── deck/               # Strategy Pitch Deck router & layouts
│   ├── favicon.ico         # Custom favicon asset
│   ├── globals.css         # Styling system (Vanilla CSS variables, responsive engine)
│   ├── icon.png            # Next.js autogenerated app icon
│   ├── layout.tsx          # Root viewport and theme context binding
│   ├── page.module.css     # Local style variables
│   └── page.tsx            # Main Explore view, listing database, and state router
├── components/
│   ├── AnywhereDoor.tsx    # Floating AI assistant drawer and prompt engine
│   └── ListingCard.tsx     # Airbnb property card component (with wishlist support)
├── context/
│   └── ThemeContext.tsx    # 3-Way theme switcher (System, Light, Dark)
└── hooks/
    └── usePropertyImage.ts # Custom image preloaders
```

## 🔌 API Documentation

### 1. Geolocation Proxy (`GET /api/geolocation`)
Returns the user's city name based on their incoming request IP. Avoids client-side browser CORS restrictions.
* **Response Format:**
```json
{
  "city": "New Delhi",
  "countryCode": "IN",
  "success": true
}
```

### 2. Pexels Image Fetcher (`GET /api/images?query=...`)
Returns a list of high-quality image URLs matching the search query. Used for mock listing properties and service imagery.
* **Headers:** Required server environment variable `PEXELS_API_KEY`.
* **Response Format:**
```json
{
  "images": [
    "https://images.pexels.com/photos/.../pexels-photo-..."
  ]
}
```

### 3. Anywhere Door Plan Generator (`POST /api/plan`)
Parses the natural language prompt and compiles the AI response matching listings, custom itineraries, and budget breakdowns.
* **Payload:**
```json
{
  "prompt": "Beachfront villa in Goa for 3 days, $150 budget"
}
```
* **Headers:** Required server environment variable `GEMINI_API_KEY`.
* **Response Format (Structured JSON):**
```json
{
  "itinerary": [
    {
      "day": 1,
      "theme": "Arrival & Beach Walk",
      "activities": ["Check into the beachfront villa", "Sunset dinner at Curlies"]
    }
  ],
  "properties": [
    {
      "title": "Stunning Beachfront Villa",
      "price": 4500,
      "rating": 4.9,
      "reviewsCount": 142,
      "pros": ["Direct beach access", "Beautiful terrace"],
      "cons": ["Slightly noisy during peak season"],
      "amenitiesMatched": ["Wifi", "Kitchen", "AC"]
    }
  ],
  "splitBreakdown": {
    "totalAmount": 13500,
    "perPersonAmount": 4500,
    "numberOfPeople": 3,
    "categories": [
      { "name": "Accommodation", "amount": 10500 },
      { "name": "Activities & Dining", "amount": 3000 }
    ]
  }
}
```

## 🎨 Styling System & Theme Controls

This module uses **Vanilla CSS** (`src/app/globals.css`) for UI control. It includes:
* **Hashed Variables:** Set up for standard design tokens (colors, font-weights, spacings, shadows).
* **React Context (`ThemeContext.tsx`):** Distributes `system`, `light`, and `dark` themes. Modifies the `[data-theme]` attribute on the `<html>` document.
* **Responsive Layout Queries:** A media query block at `min-width: 769px` toggles layout states between a native mobile shell mockup and a full-screen desktop grid with slide-out side drawers.
